Getting Set up With Visual Studio

Introduction

Microsoft Visual Studio is an application that allows user to create and develop programming projects across multiple languages in one location. Visual Studio has options for all kinds of programmers and developers. Microsoft offers at least one option across the three major Operating systems and at least two options for Windows and Mac users. The install is a personalized option for project specific users or someone looking to explore different languages. All of that information and more is housed in this section of the site, the overall goal is to compile some basic user guides to Microsoft Visual Studio, including installing the application, starting your projects and more.

See the menu below to easliy find the topic or task you are looking for.

Installing Visual Studio

Installing Visual Studio is a rather straight forward procedure. The installation page can be found by doing a general search on a browser of your choosing, just be sure to look for the licensed Microsoft page. Additionally, by navigating to the Microsoft homepage click on the 'All Microsoft' tab in the top right corner of the screeen and click on the 'Visual Studio' link under the 'Developer & IT' tab. For simplicity sake Click Here to redirect to the installation page.

The once on the installation page, you are met with multiple install options make sure to find the corresponding OS install package. The page offers install packages exculusively Windows and Mac, Microsoft also offers an option that runs on Mac, Window and Linux called 'Visual Studio Code.' Microsoft advertises this option for Java and web developers due to but, it still has plenty of oportunity for other programming languages

Once the installation has finished an application should show up in your programs folder called 'Visual Sudio Installer', find the application open it. Once opened you will be met with a window of workload options, it is important to read and understand what tools are included in the following packages as they center around certain developemental topics. If you are unsure which package best suits your needs or are concerned about the installation size, see the 'Installation details' section on the far-right side of the window or navigate to the 'Individual Componets' tab at the top of the window. In the same ribbon at the top of the window should be a tab labeled 'Installation Locations', here you can change the location that the differnt workload and component packages install to.

Starting a Project in Visual Studio

Before beging a project before familiar with the interface of Visual Studio. Depending on your programming language of choice some items may be slightly different, but overall everything should function relatively the same. Just like all other Microsoft products in the header of the window you will find a 'File', 'Edit', and 'View' along with some other options. The key tabs to note here are the File, Edit, and Help tabs, the File tab allows us to open projects using the open function and save projects using the Save and Save All functions. The Edit tab allows us to utilize tools like Undo, Redo, Cut, and Paste while just about everything in this tab is possible using keyboard shortcuts, its good to know where these and similar tools are located. Lastly, is covering the Help tab which houses Visual Studio software information and community help. Inside the Help tab you will find a 'Check for software updates' option, this is an important feature to look at on a consistant basis as keeping an update software development program can help the programmer make the most of the software. Additionally, the 'About Visual Studio' option will inform us what version we are running on and what packages we have installed. The final key feature to in the Help tab is the 'View Help' option which opens up the Microsoft hosted forums for any errors or issues that you may run into, if you find an error code in your debug tool that you don't understand it may be worth looking at the webpage for assisstance. You can view that page Here.

Another important step to setting up Visual Studio is creating a location to save all you projects. It is recommended that you create folder dedicated to Visual Studio projects that is easily accessible. Make sure you name it something memorable and it isn't burried inside a plethra of other folders, notice how my folder is labeled something simple like 'VS Projects' and that my folder location is not an long address.

Once you have created your folder and are ready to start your project, go and open up your Visual Studio Application. Once opened you will be met with a window with options to start a project, here you will want to select 'Open local folder'. You'll want to find the folder that you just created a moment ago, select the folder and hit the open button in your file explorer. At first you will be met with a blank Visual Studio workspace window, you'll want to find 'File' in the top left hand corner of the window and hover over the 'New' option and select 'File.' Upon doing so you will once again be met with another window, allowing you to select the file format of your choosing. Note that on the left-hand side of that window there are options to sort by programming language. Alternatively, in the solution explorer on the right-hand side of the Visual Studio workspace is the solutuion explorer, here you can right-click underneath your folder and hover over the 'Add' option and select 'New File.' Notice that the file does not automatically format unless specified. Here, we can name the file something of your choosing, I would recommend keeping the file name short and unique as the file will more than likely by refrenced somewhere in the program itself.

After choosing a name for your file, it is important to add the correct file extension to the end of the name. The file will be formatted as: "file-name.extension". Each language holds a unique extension that is added to the end of each file name, see the table below to see the programming language and the file extension.

Programming Language File Extension Example name
C++ .cpp example.cpp
HTML .html example.html
Java Script .js example.js
Python .py example.py

Compiling your project

Once you have completed your project, it is important to run your program and your debugger tool to make sure the program runs as intended. Each language configures slightly different which will be covered below

C++

Runnning a C++ is accessible with two options. In the ribbon at the top of your screen there should be a green play button with some text to the right that says 'Local Windows Debugger' and another green play button just to the right of that. Running the local debugger will display some diagnostic tools on the right side of the window and open a command prompt screen with your program. While the start without debugging option will open the same command prompt without the diagnostic tools oon the right side.

HTML

Running an HTML program is as easy as saving a file. Really, all you need to do is save your project in the file folder we created earlier, then you will want to locate that folder, and locate the chrome icon with the name of your file in that folder. You should be able to double-click the icon and it should automatically pull your web page up in a new tab. If you are unhappy with some of the looks of your page, you can make your changes in visual studio, save the file and refresh the chrome page, which should show the new changes

Java

Unfortunately Visual Studio will not run Javascript code on its own, you'll notice that there are similar looking play buttons in the ribbon of the visual studio window and when you click them a command prompt will show up on screen and quickly disapear. You will have to install some kind of external code editor in addition to the NodeJS software as well.

Python

Python has a nearly identical set up to C++, yet again in the ribbon of your window you will see to play buttons. Similar to C++ one option is a run without debugging tools and the other will run with debugging tools. After selecting one of the options, a command prompt window will open and display your code.
